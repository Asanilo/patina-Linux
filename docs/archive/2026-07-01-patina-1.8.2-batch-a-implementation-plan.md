# Patina 1.8.2 Batch A Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Release Patina `1.8.2` with atomic local API configuration, secure Token ownership, safe custom-category management, and expanded Linux browser-family matching.

**Architecture:** The local API Token file becomes the sole Token source while SQLite retains only the configured port; a pre-bound listener is installed only after port persistence succeeds. Classification adopts stable extended-category IDs with separate labels and transactional draft operations, restricted so only custom categories can be renamed or merged away. Browser support remains a Rust domain policy with extension-side identity used only for metadata and diagnostics.

**Tech Stack:** Rust 2021, Tokio, Tauri 2, SQLx/SQLite, React 19, TypeScript, Node test scripts, Firefox Manifest V2.

**Design:** `docs/archive/2026-07-01-patina-1.8.2-batch-a-design.md`
